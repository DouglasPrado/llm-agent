# AgentX SDK

O AgentX SDK e uma biblioteca TypeScript **standalone** e **in-process** para construir agentes de IA conversacionais. Ele oferece uma arquitetura completa para criar agentes inteligentes com streaming em tempo real, execucao de ferramentas, memoria persistente, busca por conhecimento (RAG) e sistema de skills modulares.

<Note>
O AgentX SDK funciona com **qualquer API compativel com OpenAI** — incluindo OpenRouter, OpenAI direta, Azure OpenAI, Groq, Together AI, e outros provedores.
</Note>

## Arquitetura

O SDK segue uma arquitetura em camadas com interfaces plugaveis:

```
+------------------------------------------------------+
|                    Public API                         |
|   Agent.create() → stream() / chat() / addTool()     |
+------------------------------------------------------+
|                    Core Layer                         |
|   ReactLoop → ContextBuilder → ConversationManager    |
+------------------------------------------------------+
|                  Domain Layer                          |
|   ToolExecutor  MemorySystem  KnowledgeManager        |
|   SkillManager  MCPAdapter    EmbeddingService         |
+------------------------------------------------------+
|               Infrastructure Layer                    |
|   LLMClient   SQLiteDatabase   FileSystem             |
|   (fetch)     (better-sqlite3) (node:fs)              |
+------------------------------------------------------+
```

**Fluxo principal (ReactLoop):**

1. Usuario envia mensagem via `stream()` ou `chat()`
2. Context pipeline monta o prompt: system prompt + memoria + knowledge + skills + historico
3. LLM gera resposta (streaming)
4. Se houver tool calls, o agente executa e reenvia resultados ao LLM
5. Ciclo repete ate o LLM responder sem tool calls ou atingir limites
6. Eventos sao emitidos durante todo o processo

## Stack Tecnologica

| Camada | Tecnologia | Versao |
|--------|-----------|--------|
| Linguagem | TypeScript | 5.x |
| Runtime | Node.js | 22+ |
| Validacao | Zod | 3.x |
| Persistencia | better-sqlite3 + SQLite | 3.x |
| Contratos de tools | zod-to-json-schema | 3.x |

<Info>
O SDK mantem no maximo **4 dependencias diretas** em producao. HTTP e feito via `fetch()` nativo do Node.js 22+ — sem axios, sem SDKs de LLM.
</Info>

## Principios de Design

- **Dependencias minimas** — apenas Zod, better-sqlite3 e zod-to-json-schema
- **Streaming first** — `stream()` e a API primaria; `chat()` consome `stream()` internamente
- **Interfaces plugaveis** — `VectorStore`, `ConversationStore` sao interfaces; SQLite e o default
- **Falhe explicitamente, recupere graciosamente** — retry com backoff, fallback de modelo, erros com traceId
- **Isolamento por design** — mutex por thread, timeout por tool, traceId por execucao
- **Custo como constraint** — `CostPolicy` com limites por execucao e sessao
- **Observabilidade embutida** — eventos com traceId e duration
